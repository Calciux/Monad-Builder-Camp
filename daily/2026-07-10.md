## 学到的重点

- 子 Agent 管道的有效性：当 Plan 足够精确时，Implement 一次对齐，Review 只需确认，无需修复循环
- Plan Agent（133s）产出的设计文档质量直接决定了下游效率——Implement Agent 仅用 27s 完成
- §-style prompt（角色→背景→任务→约束→验收）在最小合约任务上足够，不需要 §10 全深度
- Foundry 在 WSL2 环境下需要绕过代理直连 binaries.soliditylang.org 下载 solc
- `forge test` 的 block.timestamp 警告是标准提示，已在设计文档中覆盖（矿工 ±15 秒偏差不影响以天计的投票截止时间）
- Monad Testnet RPC 不稳定（`rpc.testnet.monad.xyz` 超时），`testnet-rpc.monad.xyz` 和 `monad-testnet.rpc.thirdweb.com` 可用
- `forge create --legacy` 在 Monad Testnet 上部署需要使用 legacy 模式（非 EIP-1559）
- 完整部署链路：forge build → forge create → cast call（read）→ cast send（write）→ 区块浏览器
